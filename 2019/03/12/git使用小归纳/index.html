git使用小归纳

  • 作为即将成为前端开发工程师，对于git的使用也是要了解的，现在将我所学习到的git的知识小结一下，以便未来肯能会用到。

git的简单介绍

  • git是一款源代码管理工具（版本控制工具），git里面的命令都是linux命名，需要有linux命令操作基础

    git基础配置

  1. 查看git版本：git --version
  2. 查看git是否配置成功：git config --list

    初始化git仓储

  • git仓库可以简单理解为文件夹
  • 命令：git init
  • 在项目目录修建打开git bash，这个仓库会存放，git对我们的项目代码举行备份的文件。
    ##将代码存储到git仓库中
  1. 将代码存储到.git仓储的门口：git add ./readme.md
  2. 把仓储门口的代码放到里面的房间中：git commit -m "说明行的文字"
  3. 查看当前的状态：git status

    存储命令补充

  4. git add ./:可以把当前目录下所有修改过的文件添加到仓储门口
  5. git commit --all -m "说明信息"：一次性将所有修改过的文件添加到版本库中

    版本回退

  6. 命令：git reset --hard Head~00代表当前版本的意思，往后以此类推，数字是几就代表回退到哪一个版本，然后在此基础上，在看日志的话，命令行就只会显示当前head指向本身和它之后的所有版本。
  7. head永远指向当前最新的版本

    git通过版本号切换版本

    git reflog：可以看到每一次切换版本的记录，可以看到所有提交的版本号

    git创建分支和切换分支

  • 默认是有一个主分支master
  1. 创建分支：git branch dev,在刚创建时dev分支里的东西和master分支里的东西是一样的
  2. 切换分支:git checkout dev，切换到指定分支，这里的切换到名为dev的分支。
  3. git branch：可以查看当前有哪些分支
  4. git merge dev:合并分支内容，把当前分支与指定分支（dev），进行合并，当前分支指的是git branch命令输出的前面有*号的分支。

    提交代码到github(当做github服务器来使用）

  • git push [地址] master:会把当前分支的内容上传到远程的master分支上
  • git pull [地址] master:会把远程分支的内容下载到本地
  • git clone [地址]:会得到远程仓库相同的内容，如果多次执行会覆盖本地的内容
