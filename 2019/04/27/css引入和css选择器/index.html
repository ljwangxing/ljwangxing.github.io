css引入和选择器

发表于 2019-04-27 | 分类于 技术

前言

时隔了那么长的时间，终于写了一个静态页面，同时也趁着这个机会吧，静态页面的相关知识复习巩固一下。在这个系列的文章中，主要涉及一些项目和我平时所积累的基础知识，不会完全覆盖所有的基础知识点。

css的三种引入方式

  1. 内链式引用

    内链式样式表就是把css代码直接现在现有的HTML标签当中。

  2. 嵌入式引用

    嵌入式演示表与内链式不同的是可以很方便的同时修改多个相同元素的样式属性。

  3. 外部式引入

    外部式css样式引入就是把css代码写一个单独的文件中，这个css样式文件以“.css”为扩展名，一般放在内。

    使用语法：<link href="style.css" rel="stylesheet" type="text/css">

    详细讲解：

    1. href=””内部的就为css文件的地址一般以英文命名，比如base.css或者style.css等等。
    2. rel=”stylesheet”type=”text/css”这是W3C的标准，固定写法不可修改。
    3. 标签的位置一般写在标签内。

      css选择器

      css元素选择器

最常见的css选择器就是元素选择器，换句话说，文档的元素就是最基本的选择器。在W3C标准中，元素选择器又称为类型选择器。

css分组

  1. 选择器分组

    假设希望h2元素和段落都有灰色，代码如下：h2,p{color:gray}。将h2和p选择器放在规则右边，然后用逗号隔开，就定义了一个规则，其右边的样式将应用到这两个选择器所引用的元素上面。逗号告诉浏览器，规则中包含两个不同的选择器。

  2. 通配符选择器

    css2引入了一种新的简单选择器，通配符选择器，显示为一个星号（）.该选择器可以与任何元素匹配，就像是一个通配符。比如代码：`{color:red}`。这个规则可以是文档中的所有元素都为红色。

    css类选择器详解

类选择器允许任何一种独立于文档元素的方式来指定样式。

  1. css类选择器

    该选择器可以单独使用，也可以与其他元素结合起来使用。类选择器可以结合元素选择器来使用。比如代码：p.more {color：red}。选择器会匹配class属性包含more的所有p元素，但是其他任何类型的元素都不匹配，无论是否含有此class属性。选择器p.more解释为：其class属性为more的所有段落。

  2. css多类选择器

    如果希望将一个特定的元素同时标记为重要（important）和（warning），就可以写成

    <p class="important waring"> this is vart important。这两个词的顺序无关紧要。假设class为important的所有元素都是粗体，而class为warning的所有元素为斜体，class中同时包含important和warning的所有元素还有一个银色的背景，就可以写作：.important{font-weight;}.warning{font-style:italic}.important.warning{background:silver}。如果一个多类选择器当中包含类名列表当中没有的一个类名，匹配就会失败。

  3. cssID选择器

    在某些方面，ID选择器类似于类选择器，但是也有些重要差别。

    语法

    首先，ID选择器前面有个#号。ID选择器不引用css属性的值，他要引用id属性中的值。与类不同，在一个HTML文档中，ID选择器会使用一次，而且仅一次。不同于类选择器，ID选择器不能结合使用，因为ID选择器不允许有以空格分隔的此列表。ID可以包含更多含义。类选择器和ID选择器可能是会区分大小写的。这取决于文档的语言，HTML和XHTML将类和ID定义为区分大小写，所有类和ID值的大小写必须与文档中的相应值匹配。

  4. css属性选择器

    css2引入了属性选择器。属性选择器可以根据元素的属性及属性值来选择权元素。

    简单属性选择器

    如果希望选择有某个属性的元素，而无论属性是什么，可以使用简单的属性选择器。如果希望把包含标题title的所有元素变为红色：*[title]{color:red}。可以只对有href属性的应用样式：a[href]{color：red}。

    根据属性值选择

    除了选择某些拥有属性的元素，还可以进一步缩小范围，只选择有特定属性值的元素。假设希望将指向web服务器上某个指定文档的超链接变成红色，可以这样写：a[href="http://www.baidu.com]{color:red}。这种格式要求必须与属性值完全匹配。

    根据部分属性值选择

    如果需要根据属性值中的词列表的某个词进行选择，则需要使用波浪号（~）。假设想选择class属性中包含important的元素，可以用代码：p[class~="important"]{color:red}

    子串匹配属性选择器

    是css2之后发布的，其中包含了更多部分值的属性选择器，按照规范的说法应该叫子串匹配属性选择器。

  5. css后代选择器

    后代选择器又称为包含选择器，可以选择作为某元素后代的元素。

    根据上下文选择元素

    我们可以定义后代选择器来创建一些规则，使这些规则在某些文档结构中起作用，而在另外一些结构中不起作用。例子：如果希望之对h1中的em元素应用样式：h1 em{color:red}。

    语法解释

    在后代选择器中，规则左边选择器一端包含两个或者多个用空格分割的选择器，选择器之间的空格是一种结合符。每个空格符可以解释为……在……找到，但是要求必须从右往左读选择器。如上面：作为h1元素后代的任何em元素。

  6. css子元素选择器

    与后代选择器相比，子元素选择器只能选择作为某元素子元素的元素。

    选择子元素

    如果不希望选择任意的后代元素，而是希望缩小范围，只选择某个元素的子元素，请使用子元素选择器。例如，现在希望选择作为h1元素子元素的strong元素：h1>strong{color:red}

  7. css相邻兄弟选择器

    相邻兄弟选择器可选择紧接在另一元素后的元素，且二者具有相同的父元素。例如：如果要在增加紧接在h1元素后出现的段落的上边距：h1+p{margin-top:50px;}。读作：选择紧接在h1元素后出现的段落，h1和p元素拥有相同的父元素。
